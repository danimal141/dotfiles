# Book Digest Skill

技術書PDFを体系的に読み解き、ポータブルな学びのドキュメント（Markdown）を生成するSkill。

## Overview

このSkillは以下のパイプラインで技術書を処理する：

1. **目次抽出** — PDFからTable of Contentsを特定・構造化
2. **全体像把握** — 書籍のスコープ、対象読者、全体アーキテクチャを要約
3. **章ごと深掘り** — 各章から「学びのポイント」を抽出しサマリ化
4. **全体まとめ** — 章横断の洞察を統合し、ポータブルな学びとして再構成
5. **Markdown出力** — Notion/Obsidian等に貼れる形式で最終成果物を生成

## Step 0: Preparation

PDFファイルの場所を確認する。ユーザーがアップロードしたファイルは `/mnt/user-data/uploads/` にある。

```bash
ls /mnt/user-data/uploads/
```

テキスト抽出に必要なライブラリをインストールする：

```bash
pip install pdfplumber --break-system-packages
```

## Step 1: 目次の抽出と構造化

まずPDFの全体構造を把握するために、最初の数十ページからテキストを抽出し、目次（Table of Contents）を特定する。

### 方針

- `pdfplumber` でPDFの最初の30ページ程度を読み、目次らしきページを見つける
- 目次が見つからない場合は、各ページの見出しパターン（"Chapter X", "第X章" 等）をスキャンして構造を推定する
- 目次の各項目を以下の形式で構造化する：
  - 章番号
  - 章タイトル
  - 開始ページ（取得できる場合）

### 実行手順

1. PDFの先頭30ページのテキストを抽出
2. "Contents", "目次", "Table of Contents" などのキーワードで目次ページを特定
3. 目次を構造化してJSON形式で保持（後続ステップで利用）

目次が特定できたら、ユーザーに目次の概要を見せて、書籍の全体構造を共有する。

## Step 2: 全体像の把握

目次から得られた構造をもとに、書籍の全体像を要約する。

以下の情報を簡潔にまとめる：
- **書籍タイトルと著者**（PDFメタデータまたは表紙ページから）
- **対象読者**: 誰に向けて書かれた本か
- **スコープ**: 何をカバーしていて、何をカバーしていないか
- **全体の構成ロジック**: なぜこの章構成になっているのか、著者の意図を推測

これを最終ドキュメントの冒頭セクションとして記載する。

## Step 3: 章ごとの深掘りサマリ

ここが最も重要なステップ。各章のテキストを抽出し、以下の3カテゴリに分けて学びを抽出する。

### 抽出する「学びのポイント」3カテゴリ

1. **設計原則・ベストプラクティス**
   - 著者が推奨するアプローチ、パターン、ルール
   - 「〜すべき」「〜してはいけない」という明確な主張
   - 例: "関数は一つのことだけを行うべき" (Clean Code)

2. **概念の定義・メンタルモデル**
   - 著者が定義・導入する重要な概念
   - 読者のものの見方を変えるフレームワーク
   - 例: "技術的負債" という概念の定義と分類

3. **著者独自の知見・洞察**
   - 他の本にはない、著者ならではの視点
   - 経験から来る具体的なアドバイス
   - 業界の常識に対する反論や独自の立場

### コンテキストウィンドウ制約への対応

技術書は数百ページになるため、全章を一度にコンテキストに入れることはできない。以下の方式で処理する：

1. 目次から章の開始・終了ページを特定（推定含む）
2. **1章ずつ**テキストを抽出し、その章のサマリを生成
3. サマリをファイルに逐次書き出し、メモリを節約
4. 章が非常に長い（50ページ超）場合は、さらにセクション単位で分割処理

### 各章のサマリフォーマット

```markdown
## 第N章: [章タイトル]

### この章の概要
[1-2文で章の目的と範囲]

### 設計原則・ベストプラクティス
- **[原則名]**: [説明]（自分の言葉で要約、著作権に配慮し引用は最小限に）

### 概念・メンタルモデル
- **[概念名]**: [定義と意義の説明]

### 著者の洞察
- [著者独自の視点や、特に印象的な主張]

### 実践への示唆
- [この章の内容を日常の開発・マネジメントにどう活かせるか]
```

重要: 著作権に配慮し、原文の直接引用は避ける。すべて自分の言葉で要約・再構成すること。

## Step 4: 全体まとめの生成

全章のサマリが揃ったら、章を横断した統合的なまとめを生成する。

### まとめに含める内容

1. **書籍全体の核心メッセージ**: この本が最も伝えたいことは何か（1-3文）
2. **章横断テーマ**: 複数の章に繰り返し現れるテーマやパターン
3. **学びのマップ**: 抽出した学びの全体的な関係性（どの概念がどの原則の基盤になっているか等）
4. **明日から使えるアクションリスト**: すぐに実践できる具体的な行動3-5個
5. **さらに深掘りすべきトピック**: この本をきっかけに調べたくなるテーマ

## Step 5: Markdown出力

最終成果物を以下の構造でMarkdownファイルとして出力する。

### ドキュメント構造

```markdown

> 著者: [著者名]
> 読了日: [今日の日付]
> ダイジェスト作成: Claude (book-digest skill)

## 全体像

[Step 2で生成した全体像]
```

## 注意事項

- **著作権**: 原文の直接引用は最小限に留め、すべて自分の言葉で要約する。コード例の転載は避け、概念の説明に留める。
- **ページ推定**: PDFの構造によっては目次のページ番号とPDFの物理ページ番号がずれることがある。目次のページ番号から実際のPDFオフセットを推定して調整する。
- **日本語・英語対応**: 書籍の言語に合わせてサマリの言語を決定する。ただしユーザーが特定の言語を指定した場合はそれに従う。
- **処理進捗**: 章ごとの処理は時間がかかるため、各章の処理開始時にユーザーに進捗を報告する（例: "第3章/全12章を処理中..."）。
